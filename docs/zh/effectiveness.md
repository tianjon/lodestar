# 为什么这套方法有效

整套设计的主线可以压缩成一句话：**不要默认相信自己最近的想法。** 下面的每个机制，都是在执行这句话。

## 外部化目标作为固定设定点

目标通常只存在于对话里，而对话会被近因侵蚀。Lodestar 把目标移到一个小而稳定的外部 artifact 中，让 agent 在行动前重新读取。固定设定点是对抗近因偏差的配重：模型可以把最新指令和活跃目标、完成定义进行比较，而不是简单追随最新输入。

## 认知科学基础

这些名字是指向概念的路标，不是诉诸权威。

- **Complementary Learning Systems**：两层结构，快速情节层 `log.md`，慢速语义层 `state.md`。consolidation 把情节细节回放成稳定结构。
- **MemGPT / virtual context paging**：有限上下文本质上是记忆层级；大小预算触发冷细节分页到 archive。
- **Generative Agents（recency × importance）**：保留不是纯近因；一个旧的蓝图级决策，应该高于一条新鲜但琐碎的备注。
- **Event Sourcing**：log 是追加式事件流，current state 是从事件流投影出来的视图。历史被追加和重新投影，而不是悄悄改写。
- **Perceptual Control Theory**：四个面向组成一个负反馈回路：设定点、测量值、误差。
- **Active Inference**：GAP 是意图世界和可行世界之间的预测误差；行动的意义是缩小这个误差。
- **Mode switching**：探索和执行是不同工作状态；`explore` 保护 sensemaking，`execute` 抵抗漂移，其他模式位于二者之间。
- **TRIZ + double-loop learning**：当要求和实践矛盾时，不只是折中，而是寻找突破；系统也允许在证据面前反问目标本身。

## GAP 驱动，证据支撑

Lodestar 不要求 agent “记得更多”。它要求 agent 缩小一个差距：要求、当前现实或实践、证据、置信度、突破解、下一步行动。实践判断必须带证据，或者明确标记 `evidence: missing`，这样流畅猜测就不能伪装成真实实践。

## hook 是强制层，但要诚实描述

协议依赖模型记得遵守。hook 在生命周期节点注入上下文，从而降低这种依赖。这不是完美控制：判断仍由模型完成。hook 提高的是**下限**：关键时刻 anchor 在场。这是实在收益，也只是 Lodestar 诚实宣称的收益。

## 诚实的限制

拒绝不可验证的宣传，本身就是方法的一部分，所以限制也要说清楚：

- 价值是**有条件的**。它最强的地方，是原生 harness 功能够不到的地方：跨会话、跨压缩、跨 harness。对于目标清晰的短单会话任务，它就是额外开销。
- 漂移检查仍然由**会漂移的同一个模型执行**；外部触发会降低自指问题，但不能消除它。
- **过期 anchor 会误导**，而且会带着书面记录的权威误导。维护纪律，以及未来把不变量变成可执行检查的 `doctor` lint，才是让系统保持诚实而不是装饰的关键。
