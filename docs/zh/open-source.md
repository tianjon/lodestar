# 开源运营说明

## 面向谁

Lodestar 面向在真实项目中使用 AI coding agent 的人：维护者、独立开发者、咨询顾问、研究者，以及在 Claude Code、Codex 和任务技能之间切换的团队。

## 项目承诺

- 保持**核心可移植**，不绑定单一 harness。
- 优先使用**可读 Markdown**，而不是隐藏运行态。
- 保持 **hook 可选、可审查**；hook 加载上下文，但不静默修改文件。
- 默认写清楚**隐私和脱敏边界**。
- 只做**诚实声明**，不说无法验证的话。

## 由理念推出的设计承诺

- **强制层属于适配器，不属于核心。** 即使 runtime 没有 hook，协议也必须仍然可用。
- **默认最小化。** 只有会改变下一步行动的结构才值得加入。
- **轻量 DDD 保持为镜头。** 统一语言和限界上下文可以有；不把 repository 和 aggregate root 强塞进代码库。

## 最值得贡献的地方

- 上手和安装清晰度；
- 跨 harness 的 hook 可靠性；
- 状态模板和 worked examples；
- 文档和翻译；
- 隐私与脱敏边界；
- 保持协议可移植的测试。

## 我们避免什么

- 把 Lodestar 变成通用数据库。
- 在没有强理由时增加运行时依赖。
- 让 hook 静默修改项目文件。
- 把私有项目状态写进公开文档。
- 做无法验证的宏大宣传。
- **让测试夹具绕开它本该捕捉的失败路径。** 一个从未触发真实失败条件的绿色测试，只是假信号；测试必须复现真实失败。
